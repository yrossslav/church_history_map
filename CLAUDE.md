# CLAUDE.md — гид по проекту «Карта истории христианства»

Руководство для ИИ-агента, продолжающего проект. Прочитай целиком перед изменениями.

## 1. Что это

Интерактивная веб-карта-генеалогия церквей и течений христианства — от апостольской Церкви
(~33 г.) до наших дней: происхождение каждой ветви, ключевые даты, эпохи, связи; у каждого
блока — описание и ссылки на источники. Это **статический сайт** (HTML+CSS+JS, без сервера):
открывается двойным кликом по `index.html` или публикуется на GitHub Pages.

Онлайн: https://yrossslav.github.io/church_history_map/

## 2. Структура

```
index.html, app.js, styles.css   — ДВИЖОК сайта (пан/зум, карточки, источники, эпохи, этапы)
data.js                          — СГЕНЕРИРОВАННЫЙ артефакт (window.DATA). Руками НЕ править!
generator/
  data/                          — ★ КОНТЕНТ (его и правят) — YAML
    branches/<ветвь>.yaml        — карточки, сгруппированные по ветви (lut, ref, rad, east, eorth, rus, pent, …)
    events.yaml                  — даты-события (название, подробность, источники)
    sources.yaml                 — библиография: ключ → {t: запись, u: URL}
    config.yaml                  — цвета ветвей, эпохи (фон), «широкие» категории, ключевые даты, геометрия
  engine.py                      — рендер: грузит data/*.yaml, строит SVG + раскладку
  build.py                       — сборка: engine → пишет ../data.js (и generator/reformation.svg для печати)
```

**Поток:** правишь YAML в `data/` → `python3 build.py` → обновлённый `data.js` → сайт показывает.

## 3. Как добавить материал (главное)

**Новая карточка (течение/церковь/движение):** добавь запись в нужный
`generator/data/branches/<ветвь>.yaml`:
```yaml
- id: lut                  # уникальный короткий id (латиница)
  title: Лютер · Меланхтон
  sub: оправдание верой    # короткая подпись (необязательно)
  column: 1                # положение по горизонтали (см. §5)
  year: 1520               # положение по времени
  branch: lut              # цвет/ветвь (ключ из config.yaml→colors)
  info: >
    Полное описание карточки (обычный текст, без markdown).
  sources: [macculloch, schaff7, needham3]   # ключи из sources.yaml
  wide: [1, 1]             # необязательно: растянуть на колонки (категория-«зонтик»)
  live: true              # необязательно: ветвь жива доныне (пунктир к низу)
  edges:                   # связи, ИСХОДЯЩИЕ из этой карточки
    - to: lutc
      type: solid          # solid | dash
      label: ""            # подпись на стрелке (необязательно)
      info: "Описание связи для карточки (необязательно)"
```
Файл можно выбрать по смыслу ветви; id уникален глобально. Связь живёт в файле «родителя».

**Новая дата-событие:** добавь в `data/events.yaml`:
```yaml
1648:
  name: Вестфальский мир — конец религиозных войн
  key: true                # выделить золотой полосой
  info: Подробное описание для карточки (необязательно)
  sources: [sannikov, needham3]
```

**Новый источник:** добавь в `data/sources.yaml`: `ключ: {t: "полная запись", u: "URL или пусто"}`.
URL обязательно проверяй (web_fetch); битый/404 — недопустимо, тогда `u: ""`.

**Почти вся структура — в `data/config.yaml`:** цвета ветвей (`colors`), эпохи-фон (`periods`),
«зонтики»-категории (`wide`), геометрия (`geometry`), группы/подгруппы (`groups`/`subgroups`),
заголовки этапов (`stages`), подписи и описания бантов/групп (`band_children`, `band_info`,
`hus_band`, `group_info`, `group_src`), ключевые даты (`key`). В коде (`engine.py`) осталась только
логика рендера: расположение больших бантов (Апостольская/Восточная/Западная/Реформация), Крест и
ручные связи-расколы. Правило: «весь контент — в `data/`, вся структура — в `config.yaml`, код —
только рендер».

После правок — обязательно пересобрать и проверить (§4, §6).

## 4. Пересборка

```bash
cd generator
python3 build.py        # читает data/*.yaml, пишет ../data.js
```
Зависимости: `python3`, `pyyaml`, `pillow` (`pip install pyyaml pillow --break-system-packages`).
Для проверки рендером — `cairosvg`. Шрифт ищется автоматически (DejaVu/Arial).

## 5. Система координат

- **Год** → вертикаль. Ось ПРОПОРЦИОНАЛЬНАЯ, но с минимальным зазором между соседними годами
  (плотные эпохи не наезжают). См. блок `_ypos` в `engine.py`.
- **Колонка** → горизонталь; бывает ОТРИЦАТЕЛЬНОЙ (Восток слева).
- **Правило:** у связи год родителя СТРОГО меньше года потомка (иначе стрелка вверх, выглядит
  оторванной). Пара (колонка, год) уникальна.

Раскладка колонок:
```
Восток (отрицательные): -6 Церковь Востока (несториане) · -5 дохалкидонские · -4 восточные секты ·
   -3 славянские/русские ветви · -2 греко-византийское православие · -1 русская линия
Запад/ствол (0+): 0 ствол (Апостольская-плашка, Доникейская, Имперская) + предшественники + католичество(16) ;
   1 раннее монашество/Августин, средневек. папство, лютеранство ; 2–5 ереси, ордена/схоластика, реформатство ;
   4–8 радикальная Реформация ; 9–11 английская линия ; 12–13 баптисты ; 14–15 ЕХБ ; 6–13 Новое время
```

## 6. Проверка (обязательно после правок)

`build.py` сам падает на грубых ошибках. Дополнительно отрендери и проверь:
```bash
cd generator && python3 build.py
python3 - <<'PY'
import json,cairosvg
s=open("../data.js",encoding="utf-8").read(); D=json.loads(s[len("window.DATA="):].rstrip().rstrip(";"))
ym={n["id"]:n["y"] for n in D["nodes"]}
up=[(a,b) for a,b in D["edges"] if a in ym and b in ym and ym[a]>ym[b]+5]   # «обратные» стрелки
col={}
for n in D["nodes"]: col.setdefault(round(n["x"]),[]).append((n["y"],n["y"]+n["h"],n["id"]))
ov=[]
for k,v in col.items():
    v.sort()
    for i in range(1,len(v)):
        if v[i][0]<v[i-1][1]-2: ov.append((v[i-1][2],v[i][2]))   # наложения боксов
print("nodes",len(D["nodes"]),"| up-edges",up,"| overlaps",ov[:10])
W,H=D["W"],D["H"]
open("/tmp/c.svg","w",encoding="utf-8").write(f'<svg xmlns="http://www.w3.org/2000/svg" width="{W}" height="{H}" viewBox="0 0 {W} {H}"><rect width="{W}" height="{H}" fill="#F7F1E3"/>{D["svg"]}</svg>')
cairosvg.svg2png(url="/tmp/c.svg",write_to="/tmp/c.png",output_width=1500)
PY
```
`up-edges` и `overlaps` должны быть пусты. Открой `/tmp/c.png`.

## 8. Источники (контент)

Двуреченский «История Реформации» (рукопись), С. В. Санников «Двадцать веков христианства» (т.1–2),
Ф. Шафф, N. Needham «2000 Years of Christ's Power» (vol. 3–4), А. Макграт, У. Эстеп и др.
Полные записи и онлайн-ссылки — в `data/sources.yaml` и в карточках («Все источники» в меню).
Тексты-первоисточники в репозитории НЕ лежат — они уже отражены в `info` карточек.

## 10. Конвенции

- Контент — на русском; `info` — обычный текст без markdown (карточка показывает plain text).
- Не дублируй id; не делай собор/событие карточкой (события — в `events.yaml`); год родителя < года потомка.
- Онлайн-ссылки в `sources.yaml` проверяй (web_fetch); не выдумывай URL — лучше пустой `u`.
- После правок всегда пересобирай `data.js` (`build.py`) и проверяй рендером.
