# CLAUDE.md — гид по проекту «Карта истории христианства»

Это руководство для ИИ-агента, продолжающего работу над проектом. Прочитай его целиком
перед изменениями.

## 1. Что это

Интерактивная веб-карта-генеалогия церквей и течений христианства — от апостольской Церкви
(~33 г.) до наших дней. Показывает происхождение каждой ветви, ключевые даты, эпохи, связи;
у каждого блока — описание и ссылки на источники. Цель — исследовательский инструмент,
которым можно делиться (опубликовано на GitHub Pages).

**Это полностью статический сайт** (HTML + CSS + JS, без сервера). Открывается двойным
кликом по `index.html` или публикуется как есть.

## 2. Структура репозитория

```
index.html      — оболочка страницы (DOM: #viewport, #topbar, #axisbar, #minimap, #menu, #card …)
styles.css      — оформление
app.js          — ДВИЖОК (пан/зум, карточки, источники, залипающие шапка/таймлайн, эпохи, миникарта)
data.js         — ДАННЫЕ карты (СГЕНЕРИРОВАННЫЙ артефакт: window.DATA = {...})
README.md       — короткое описание для пользователя
CLAUDE.md       — этот файл
generator/      — ИСХОДНИК карты (из него собирается data.js)
   timeline5.py   — ЕДИНЫЙ ИСТОЧНИК ПРАВДЫ: все узлы, связи, описания, события, геометрия
   enrich_data.py — переопределения/дополнения описаний и источников (накладываются на timeline5)
   build_data.py  — сборщик: импортирует timeline5 → пишет ../data.js (и generator/reformation.svg)
```

> **Важно:** `data.js` руками НЕ редактируют. Правят `generator/timeline5.py`
> (и `generator/enrich_data.py`), затем пересобирают (см. §4).

## 3. Архитектура

Данные отделены от движка. `app.js` ничего не знает о содержании — он рисует то, что лежит
в `window.DATA`. Поэтому расширение материала = правка данных + пересборка, без правки движка.

`window.DATA` содержит:
- `svg` — внутренний SVG схемы (боксы-узлы, линии-связи, банты, плашки);
- `nodes[]` — `{id,x,y,w,h,color,group}` (для миникарты и подсветки);
- `edges[]` — пары `[a,b]` id (для подсветки ветви по клику);
- `events[]` — `{year,yy,key,text,info,src}` (левый таймлайн);
- `headers[]` — `{label,key,color,stage,x0,x1}` (заголовки шапки по этапам);
- `stageBounds`, `stageNames` — границы и названия трёх этапов;
- `periods[]` — `{label,yy0,yy1,alt}` (фон-эпохи);
- `groups[]` — колонки-ветви (для вертикальных разделителей);
- `bib{}` — `ключ → {t:"библ.запись", u:"URL или пусто"}` (источники);
- `W,H,cx0,cx1,cy0,cy1,bandY,rs` — геометрия/границы.

Узлы/связи/карточки в `svg` помечены классами `it node` / `it edge` и атрибутами
`data-id`, `data-t` (заголовок), `data-i` (описание), `data-c` (цвет), `data-src` (ключи
источников через запятую). Движок по ним строит карточки и подсветку.

## 4. Как пересобрать data.js

Из каталога `generator/`:
```bash
cd generator
python3 build_data.py      # пишет ../data.js и generator/reformation.svg
```
Зависимости: `python3`, `pillow` (`pip install pillow --break-system-packages`).
Шрифт ищется автоматически (DejaVu/Arial); если не найден — рисунок соберётся, но переносы
текста будут чуть грубее (на сам data.js не влияет критично).

После сборки полезно проверить движок: `node --check app.js` (если есть Node).

## 5. Система координат (ВАЖНО для расположения)

В `timeline5.py` каждый узел задаётся как `id: (КОЛОНКА, ГОД, "ветвь", "Заголовок", "подпись")`.

- **Год** → вертикаль. Ось ПРОПОРЦИОНАЛЬНАЯ: `yof(y)=TOP+(y-Y0)*PXY`, но с минимальным
  зазором `MINGAP` между соседними годами (плотные эпохи не наезжают). См. блок вычисления
  `_ypos` в timeline5.
- **Колонка** → горизонталь: `colx(c)=COLX0+c*CW+CW/2`. Колонки могут быть ОТРИЦАТЕЛЬНЫМИ.
- **Правило непротиворечивости:** у связи `(a,b)` год родителя `a` должен быть МЕНЬШЕ года
  потомка `b` (иначе стрелка пойдёт вверх и будет выглядеть оторванной). Всегда проверяй.
- Пара (колонка, год) должна быть УНИКАЛЬНОЙ (иначе боксы наложатся).

Раскладка колонок (положение = столбец; столбцы переиспользуются в разные эпохи):
```
ВОСТОК (отрицательные):
 -6 Церковь Востока (несториане)      -3 славянские/русские ветви (Кирилл-Мефодий, Болгария,
 -5 дохалкидонские (монофизиты)            Брестская уния, Могила, старообрядцы)
 -4 восточные секты (павликиане,       -2 греко-византийское православие (Византия, Фотий,
     богомилы, иконоборчество, унии)        исихазм, падение Царьграда, Иеремия II, Лукарис)
                                       -1 русская линия (Крещение Руси → автокефалия →
                                            патриаршество → РПЦ)
ОБЩИЙ СТВОЛ / ЗАПАД (0 и положительные):
  0  ствол: Апостольская (плашка), Доникейская, Имперская; предшественники (Уиклиф, Гус,
     Богемские братья, Эразм…); Католичество (контр-реф. — кол.16)
  1  раннее монашество/Августин; средневек. папство (rcc); Лютеранство
  2–5 ранние ереси; монашество/схоластика/ордена/вальденсы/катары; (в Реформации — реформатство)
  4–8 радикальная Реформация ; 9–11 английская линия ; 12–13 баптисты ;
  14–15 ЕХБ ; 16 Контрреформация
```
Цвета — словарь `COL` в timeline5: `anc` (золото, древняя Церковь), `ext` (серый, угасшие
ереси/течения), `east`/`eorth`/`rus` (синие, Восток), `root` (серо-стальной, католический
Запад), `pre/lut/ref/rad/eng/bap/ehb/contra` (Реформация и её ветви).

## 6. Большие «банты»/плашки и этапы

- Широкие плашки-линии: «АПОСТОЛЬСКАЯ ЦЕРКОВЬ» (вверху, во всю ширину), «Восточная
  (Православная) Церковь» и «Западная Церковь» (наравне, на уровне `RKC_YEAR=680`),
  «РЕФОРМАЦИЯ · 1517». Рисуются в сборке `DIAG` (timeline5) и спец-обработкой `apost` в боксах.
- Над всем — Крест (Христос), от него идёт ствол.
- Соборы/расколы — это СОБЫТИЯ (на левом таймлайне `EVENTS` + фон-эпохи `PERIODS`),
  а НЕ узлы. Узлы = только течения/церкви/движения. Не делай собор узлом.
- Три ЭТАПА с меняющимися заголовками шапки задаются в `build_data.py` → `STAGEHEAD`
  (I — до 1054, II — 1054–1517, III — с 1517). Движок показывает набор заголовков того
  этапа, который сейчас вверху экрана, и индикатор этапа слева внизу.

## 7. Как добавить материал (рецепты)

**Новый узел (течение/церковь):** в `timeline5.py`
1. `N["id"]=(колонка, год, "ветвь", "Заголовок", "подпись")` — выбери свободную (колонка,год),
   родитель по году выше потомка.
2. `INFO["id"]="описание 1–4 предложения, обычный текст, без markdown"`.
3. `SRC["id"]=["ключ1","ключ2"]` — ключи из `BIB` (или добавь в `BIB` новый источник
   `"ключ":{"t":"полная запись","u":"РАБОЧИЙ URL или пусто"}`; URL ПРОВЕРЯЙ через web_fetch,
   битые/404 — недопустимы, тогда `u=""`).
4. Связь: `E.append(("родитель","id","solid|dash","подпись стрелки"))`.
5. Если течение живёт доныне — добавь id в `LIVE` (протянется пунктиром к низу).

**Новая дата-событие:** `EVENTS[год]="краткое название"` и при желании
`EVENT_INFO[год]="подробное описание для карточки"`. Ключевые — добавь в `KEY`.

**Новый источник:** добавь в `BIB` (см. выше) и ссылайся ключом в `SRC`.

После любых правок — пересобери (`python3 build_data.py`) и проверь рендером (§8).

## 8. Проверка вёрстки (обязательно)

Карта большая; «на глаз» в коде не видно наложений. После правок отрендери ядро в PNG и
посмотри:
```bash
cd generator && python3 build_data.py
python3 - <<'PY'
import json,cairosvg
s=open("../data.js",encoding="utf-8").read()
D=json.loads(s[len("window.DATA="):].rstrip().rstrip(";"))
W,H=D["W"],D["H"]
open("/tmp/core.svg","w",encoding="utf-8").write(
 f'<svg xmlns="http://www.w3.org/2000/svg" width="{W}" height="{H}" viewBox="0 0 {W} {H}"><rect width="{W}" height="{H}" fill="#F7F1E3"/>{D["svg"]}</svg>')
cairosvg.svg2png(url="/tmp/core.svg",write_to="/tmp/core.png",output_width=1500)
print("ok",W,H,len(D["nodes"]),"nodes")
PY
```
(нужен `cairosvg`: `pip install cairosvg --break-system-packages`). Открой `/tmp/core.png`.
Проверяй: нет наложений боксов; все связи идут СВЕРХУ ВНИЗ; новые узлы прикреплены.
Доп. контроль направления связей — все пары `edges` должны иметь y(родителя) < y(потомка).

## 10. Источники (контент)

Содержание собрано по: учебнику истории Реформации Двуреченского (рукопись), С. В. Санникову
«Двадцать веков христианства» (т.1–2), Ф. Шаффу «История христианской церкви», N. Needham
«2000 Years of Christ's Power» (vol. 3–4), А. Макграту, У. Эстепу и др. Полные записи и
онлайн-ссылки — в `BIB` (timeline5/enrich_data) и в карточках («Все источники» в меню).
Тексты-источники (Санников .txt, главы учебника) в репозитории НЕ лежат — они уже отражены
в `INFO`. Для нового наполнения их перечитывают отдельно (через инструменты Read), затем
переносят факты в `INFO`/`SRC`.

## 12. Конвенции

- Контент — на русском; в `INFO` — обычный текст (без markdown/списков): карточка показывает plain text.
- Не дублируй id. Не делай собор узлом. Год родителя < года потомка.
- Онлайн-ссылки в `BIB` обязательно проверяй (web_fetch); не выдумывай URL — лучше пустой `u`.
- После правок всегда пересобирай `data.js` и проверяй рендером.
